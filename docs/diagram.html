Diagrama de Fluxo do OpenVAS Agent

Início da Aplicação

1. Execução do main.py

2. Exibição da arte ASCII (art.py)

3. Entrada do Usuário (Query)

Agente Supervisor

Recebe a query e decide a rota.

Função: router_function

Lógica: O LLM analisa a intenção do usuário.

Roteamento Condicional
[Criar, Iniciar]

Agente TaskCreator

Ferramenta: create_openvas_task

Cria e inicia um scan no OpenVAS.

Tool: GVMWorkflow

Interage com o usuário para obter detalhes da tarefa (alvo, nome, etc.).

[Analisar, Ver]

Agente ResultAnalyzer

Ferramenta: get_openvas_results

Busca e analisa os resultados do scan.

Tool: ResultManager

Interage com o usuário para identificar a tarefa e busca os resultados (XML).

Resultado da Ferramenta (ToolMessage)

Retorno ao Supervisor

Recebe o ToolMessage.

Lógica: Como uma ferramenta foi executada, o ciclo é finalizado.

FINISH

Saída Final

O resultado final é exibido para o usuário no console.

(Ex: "Tarefa criada com sucesso" ou a análise da vulnerabilidade).
